## A 群：v1 公開面の積み残し（→ ADR-0033 案F）

### F-1 OAuth 公開面 `porters.auth.*` が存在しない

- 約束: [ADR-0007][0007] が `authorizationUrl` / `exchangeAuthorizationCode` / `revoke`（ブラウザ `code` フロー・SD-3）＋ `ensureAuthenticated` / `getToken`（SD-6）を v1 公開面として決定。
- 実装: `src/auth/` は `code_direct` のみ。`PortersClient` に `auth` 名前空間なし（`src/client.ts:69-131`）。
- 影響: 初回の人間ブラウザ権限付与ができない＝MCP/対話シナリオ（案A）の前提を欠く。

### F-2 Read クエリ `order` / `keywords` / `itemstate` 未実装

- 約束: [ADR-0005][0005]:54-57・R-5（`docs/design/requirements.md:70`）・`docs/design/basic-design.md:69-71`、reference `docs/reference/resource-api/README.md:17-28`。
- 実装: `SearchQuery`（`src/resources/resource.ts:35-46`）は `field`/`condition`/`count`/`start` のみ。`buildReadUrl`（`:144-161`）も同 4 種のみ送出。
- 影響: 並び順・フリーワード検索が不可。**`itemstate` は削除 API 非提供下で削除済みデータを読む唯一の正規手段**（`docs/reference/resource-api/resources-list.md:47-48`）で、その経路が塞がっている。typed `condition`（`{ ge: … }`）も未実装＝raw `Record<string,string>` で到達は可（discrepancy・低）。

### F-3 `tenant(id)` ＋ per-call `partition` 上書き 未実装

- 約束: [ADR-0008][0008]:46-47・[ADR-0021][0021]（`partition(id)`→`tenant(id)` 改名）・`docs/design/basic-design.md:63,82`。
- 実装: `PortersClient` に `tenant()` なし。partition は構築時固定（`deps.partition`・`src/resources/resource.ts:175,184,187`）、呼び出し毎引数なし。
- 注: [ADR-0021][0021]:123 が「スコープ関数は未実装」と自認＝既知の未建設。per-call の**偽宣言**は RV-10 へ。

### F-4 一括書き込み（200 件バッチ＋200 超の自動分割）未実装

- 約束: `CLAUDE.md`（200 件超は分割）・reference `docs/reference/resource-api/write-format.md:24-25`（最大 200 件・create/update 混在可）。
- 実装: encoder は配列対応（`src/xml/encode.ts:108-121`）だが公開 API は単件のみ（`write` が `items:[item]`・`src/resources/resource.ts:217-238`）。`createMany`/自動分割なし（分割は requester のエラー hint 文字列だけ・`src/http/requester.ts:86`）。
- 補足（latent）: Image 型フィールド書き込みも未対応（`encodeField` に Image ケースなし）。現 MVP カタログに Image 項目なしで実害は今は無い。
